다음과 같은 작업을 수행하기위해, 각 단계별 sub agent를 `.claude/agents` 경로에 생성하세요.
각 sub agent의 보고서는 보고서 경로에 작성됩니다.

각 sub agent 파일은 다음과 같은 메타데이터를 포함해야 합니다:

```md
---
name: agent-name
description: agent description
---
```

각 sub agent 이름 형식은 다음과 같습니다.: `design-guide-{step-number}-{step-name}`

입력:

- 레퍼런스 페이지
  - https://www.claude.com/product/overview
  -

최종 목표:

- 주어진 레퍼런스 페이지의
- 보고서 경로는 `./agent-outputs/dashboard-improve/` 이다.
- 각 sub agent는 페이지와 보고서 경로를 입력받아 미리 정의된 작업을 수행한다.

1. 주어진 페이지의 내용 구성 및 UI 디자인을 파악하고, 개선해야할 부분을 엄격하게 피드백해 보고서 경로에 `0-explore.md` 파일명의 보고서로 작성한다. 디자인은 페이지 내용 구성, 참고할만한 레퍼런스, UI 디자인 컨셉, 섹션별 컴포넌트 명세, 애니메이션(framer-motion 사용)을 모두 정의해야만한다. claude.ai 랜딩페이지를 참고한다.
2. 주어진 페이지의 내용 구성 및 UI 디자인을 파악하고, 1번 단계에서 작성한 보고서를 참고하여 더 나은 계획으로 작성할 수 없을지 엄격하게 피드백한다.
   개선된 결과를 보고서 경로에 `1-plan-critic.md` 파일명의 보고서로 작성한다.
3. 코드베이스의 특징을 파악한 뒤, 2번 단계의 결과물을 토대로 구체적인 구현 계획을 작성한다. 보고서 경로에 `2-implement-plan.md`로 작성한다.
4. 3번 단계에서 작성한 보고서를 참고하여 2번 단계의 결과물을 구현하기위한 계획으로서 오류가 없는지, 빠트린 내용은 없는지 엄격하게 피드백한다. 개선된 결과를 보고서 경로에 `3-implement-plan-final.md`로 작성한다.
5. 4번 단계에서 작성된 최종 보고서를 `implement` agent를 사용해서 정확히 구현한다.
6. 5번 단계를 마친 뒤, 다음 4가지 agent를 병렬로 호출한다. 각각의 결과를 보고서 경로에 `4-check-{name}.md` 파일로 작성한다. 각 agent는 코드는 수정하지 않는다. 이미 충분히 훌륭하면 파일을 생성하지 않아도 된다.

- ui 개선이 필요한 부분을 파악해 세세히 피드백 남긴다. 전문적인 수준의 saas 랜딩페이지로서, claude.ai 랜딩페이지가 잘 참고되었는지 파악해야한다.
- typecheck, lint, build 에러가 없는지 점검한다.
- 기존 코드베이스 구조를 잘 준수하여 클린코드로 작성되었는지 점검한다.
- i18n 대응이 빠진 부분은 없는지 점검한다.

7. 6번 단계에서 생성된 모든 문서들을 참고해서 코드를 개선한다.
8. 6번 단계에서 호출한 agent들을 또 호출한다. 각각의 결과를 보고서 경로에 `5-check-{name}.md` 파일로 작성한다. 이미 충분히 훌륭하면 파일을 생성하지 않아도 된다.
